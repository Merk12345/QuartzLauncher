Developer Center

Build for Quartz

The Quartz Developer Center is for mod creators who want to build, package, test, and submit Quartz-native mods as the ecosystem grows.

Getting Started

Learn what Quartz packages are, how the launcher reads them, and what files a basic mod should include.

Mod Submission Rules

Mods should be safe, original, clearly described, and packaged correctly. Broken, unsafe, stolen, spam, or misleading packages may be declined.

Quartz Package Format

Quartz packages use a manifest-based format so the launcher can read the mod name, ID, version, author, engine, files, and entry point.

How to Package a Mod

A valid package should include a manifest, the required payload files, a clear folder structure, and any assets needed for the mod to run.

How Mod Review Works

Submitted mods may be checked for install issues, missing files, bad manifests, crashes, unsafe behavior, ownership problems, and basic quality.

Why Mods Get Declined

A decline does not always mean the mod is banned forever. It usually means something needs to be fixed before the mod is ready.

Quartz-Native Mods

Quartz-native mods are the long-term goal for the launcher, giving creators a package system built specifically around Quartz.

Developer Tools

Future dev tools will focus on creating packages, validating manifests, testing installs, exporting logs, and making mod creation easier for new devs.

Common reasons a mod may be declined

  • The package is broken or cannot install correctly.
  • The manifest is missing, invalid, or has unclear information.
  • Required files are missing from the package.
  • The mod crashes Quartz or Geometry Dash during testing.
  • The mod includes unsafe code, suspicious behavior, or harmful files.
  • The mod uses stolen work, stolen assets, or does not credit required creators.
  • The description is misleading, empty, or does not explain what the mod does.
  • The mod is spam, duplicated, low-effort, or not ready for public release.
  • The mod breaks Quartz community rules or submission rules.

Developers can fix the issue and resubmit when the mod is ready.
